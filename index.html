upd3301 / PC-8801 emulator

NEC μPD3301 CRTC を起点に、PC-8001 / PC-8801 をまるごとブラウザで動かす純JavaScriptエミュレータ。依存ゼロ・決定論・ヘッドレステスト可能。Z80×2、μPD765 FDD、YM2203/YM2608(OPNA) まで。

▶ PC-8801 を起動する μPD3301 テキストデモ GitHub

ROMは同梱していません（後述のBYO-ROM）。ブラウザ内だけで動き、データはどこにも送信されません。

何ができる — ブラウザだけで、遊ぶ・作る・改造する

  • PC-8801 (N88-BASIC) をディスクから起動（FDDサブCPU＝2個目のZ80まで実装）
  • PC-8001 (N-BASIC) 起動、μPD3301＋μPD8257 DMA の実挙動
  • フォルダを雑に指定するだけ — ROM一式・font・数百本のD88ゲームを自動スキャン＆記憶（File System Access。ファイルは端末に残る）
  • サウンド: YM2203(OPN) FM3+SSG3、サウンドボードII (YM2608/OPNA) FM6＋リズム
  • ジョグシャトル: ビデオデッキ風に巻き戻し／早送り。現在を追い越すと実エミュを×16で爆走、非破壊シークでいつでも未来へ
  • セッション自動セーブ＆再開: うっかりリセットやタブを閉じても「前回の続きから再開」
  • CRTシミュレーション: 蛍光体残光・アパーチャ/シャドウマスク（高精細ピッチ）・走査線・アナログ色。クリーン表示と切替
  • ICE / IDE: 条件ブレーク・時間旅行・プロファイラ・Z80マクロアセンブラ・逆アセン。パネルはtmux式に列で自由配置＋別ウィンドウ切り離し
  • 速度 ×0.25〜×16 / WebM録画 / 巻き戻し10分バッファ
  • 永続: ROM・ディスク・設定・進行はブラウザ(IndexedDB)に保存。閉じても残る
  • 決定論: 同seed＋同入力→同出力。`node --test` で全数検証。データはどこにも送信しない

使い方 — いちばん簡単な始め方

著作権上、NECのROMやゲームは同梱していません（自分の吸い出しを使ってください）。PC-8801デモを開いて:

  1. 💽ディスク → 📚一覧 → 📁フォルダを開く で、ROMやD88の入ったフォルダを雑に指定するだけ（例 C:\var\emulator）
  2. フォルダを再帰スキャンして PC-8801 BIOS一式・font・数百本のゲームを全部見つけ、検索付き一覧に並べる（一度許可すれば次回も記憶）
  3. 一覧から遊びたいゲームをクリック → リセットで起動。進行は自動セーブされ、次回は「▶再開」で続きから

ファイルはあなたの端末に残ったまま（ハンドルだけ保持。著作物をどこにもアップしない設計）。個別に「📁ファイル選択」でROM/ディスクを選ぶ従来方式もOK。ローカルに roms/<機種>/ を置いて manifest.json に "n88": "<dir>" を書けば、開くだけで自動ロードもできます。
※フォルダ選択は Chrome系ブラウザで動作（File System Access API）。

技術メモ

  • 純JS・依存ゼロ: レジスタin → サンプル/ピクセルout
  • 決定論: Math.random不使用、状態はすべてスナップショット可能
  • 2個のZ80: 本体CPU＋FDDサブCPUが8255ペアでハンドシェイク
  • OPN/OPNA: LOG-SIN→EXP の本物の信号経路、ボード出力段まで再現
  • ヘッドレステスト: node --test（250+テスト）
