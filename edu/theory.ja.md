# 電子ビームと競走する — CRTコントローラの基礎理論

**ユウ**：ねえ、昔のパソコンって「文字を画面に出すチップ」が別にあったって
ほんと？　文字くらいCPUが出せばよくない？

**僕**：それがさ、出せないんだよ。僕も最初「forループで画面に書けばいいじゃん」
って思ってたんだけど、ブラウン管の仕組みを知って考えが変わった。ブラウン管
ってのは電子ビームが画面の左上から右へ、1本ずつ下に降りながら光らせていく。
で、このビーム、**止まってくれない**。1秒に60回、画面全体を塗り直す。
1本の走査線はたった63.5マイクロ秒。CPUがのんびりしてる暇はないんだ。

**ユウ**：つまり、ビームが「今ここ通るよ！」ってタイミングに合わせて、
表示したいドットを差し出し続けなきゃいけない？

**僕**：そう、永遠にね。これを「ビームとの競走」って呼んでる。で、その競走を
CPUの代わりに走ってくれる専用チップがCRTコントローラ、今回の主役
**μPD3301**。PC-8001っていう1979年のパソコンに入ってた。

**ユウ**：そのチップが文字の形を覚えてるの？

**僕**：いや、役割分担が面白いんだ。文字の形（フォント）は別のROMに入ってて、
μPD3301が覚えてるのは「何行目の何桁目に、どの文字コードがあるか」の管理
だけ。しかも実は**それすら覚えてない**。文字コードの表はメインメモリに
置いてあって、μPD3301は自分でメモリを読む能力がない。

**ユウ**：え、じゃあどうやって表示するの？

**僕**：相棒がいる。**μPD8257**っていうDMAコントローラ。DMAは
「Direct Memory Access」、CPUを介さずにメモリを読み書きする仕組み。
μPD3301は「次の行のデータちょうだい！」って信号（DRQ）を上げるだけ。
するとμPD8257がメモリから1行分＝**120バイト**をドバッと運んでくる。
80文字分の文字コード＋40バイトのおまけつきで。

**ユウ**：おまけ？

**僕**：**アトリビュート**。「桁位置と属性」のペアが20組。「30桁目から黄色ね」
「50桁目から点滅ね」みたいな**付箋**なんだ。全80桁ぶんの色を毎回送ると
無駄だから、「変わり目」だけ送る。付箋の効果は次の付箋まで続く。
1行に20枚しか貼れないから、PC-8001は「1行に色を変えられるのは20回まで」
っていう制限があった。仕様書のバグじゃなくて、DMAの節約なんだよ。

**ユウ**：へー！　で、文字の形はいつ出てくるの？

**僕**：ビームが走る直前。文字コード（例えば`A`＝65）とビームの現在の行から、
キャラクタジェネレータROMの「65番の文字の上から3ライン目」みたいに引くと、
8ドット分のオン/オフパターンが出てくる。1文字は横8ドット×縦8ライン
だから、同じ文字コードを8本の走査線で使い回す。だからμPD3301の中には
1行分のバッファが2本あって、片方で表示しながらもう片方に次の行をDMAで
読み込む。ダブルバッファってやつ。

**ユウ**：さっきの付箋で「セミグラフィック」ってのも見たけど、あれは？

**僕**：付箋に「この区間は文字じゃなくてブロック絵ね」って書けるんだ。すると
文字コードの8ビットが、フォントの番号じゃなくて**2×4のタイル絵**として
読まれる。1文字のマスを横2×縦4の8ブロックに割って、ビットが1のブロックを
塗る。たった1バイトで簡易お絵かきができる。ファミコンより前の時代、
これがPC-8001の「グラフィック」だった。デモのビールジョッキもこれで
描いてる。

**ユウ**：40桁モードと80桁モードの切り替えは？　画面の広さが変わるの？

**僕**：画面の広さは同じ。変わるのは**ドットを送る速さ**。40桁モードでは
文字クロックを半分にして、1ドットを2倍の幅で塗る。μPD3301自身には
「40桁モード」なんて機能はなくて、「1行は40文字です」って設定し直すのと、
外の回路がクロックを半分にするのの合わせ技。太文字になるのはそのせい。

**ユウ**：ちなみにこの方式、なんか弱点ないの？

**僕**：あるよ、しかも有名なやつが。DMAがメモリを使ってる間、**CPUは
止まる**。PC-8001は画面を表示してるだけでCPUが3割くらい遅くなった。
だからゲームで速度が欲しい時は「画面を消して計算する」なんて荒技もあった。
逆にこの仕組みを**逆手に取った**技もある。DMAに「1画面ぶん」じゃなくて
「2画面ぶん運べ」って設定すると、2枚の画面が1フレームごとに交互に表示
される。8色しか出ないパソコンで、2色を高速で点滅させて中間色に見せる —
雑誌に載った「27色表示」の技。ハードの隙間に潜り込む遊びだね。

**ユウ**：チップの「仕様」じゃなくて「仕組みの隙間」が遊び場になるんだ。

**僕**：そこが一番好きなとこ。仕様書どおりに使えば80×25の文字が出るだけの
チップが、DMAカウンタひとつで27色マシンになる。作った人も想定してない
未来がバスの上に転がってた。

この話が分かると `index.js` の読み方が変わる：`stepFrame()` の中の
`drq(buf)` が「ビームとの競走のためにμPD8257へ行データを要求する瞬間」で、
`expandAttrRow()` が「付箋を1桁ずつの色に引き伸ばす作業」。`pc8001.js` の
`renderScreen()` がキャラジェネROM（CGROM）を引いてドットにする所、
`upd8257.js` の `drqPull()` のオートロードが27色技の種、だよ。
