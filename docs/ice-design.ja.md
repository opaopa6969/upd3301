# ICE 設計 — デバッガ・アセンブラ・石器一式

実機時代のICE（In-Circuit Emulator）が本体より高かった問題を、2026年に
タダで清算する。対象は PC-8001 / PC-8801（メインZ80＋FDDサブ基板のZ80の
両方）。UIは `demo/ice.html`（別ウィンドウ、`window.opener.__machine` を
掴む）、道具はすべて repo 直下の純粋モジュール。

## 原則

- **coreは編集しない**。計装は全部「外から」— `cpu.step` / `stepFrame` /
  busコールバックのwrapで実現する。coreが提供するのは汎用API
  （`getState`/`setState`、`snapshot`/`restore`）だけ
- **決定論を武器にする**。時間旅行も等価性検証も、同入力→同結果という
  repoの鉄則の配当。`Math.random` 禁止はUI層（フリッカー等）にも及ぶ
- **正直な尻尾**。観測できなかったものは「未分類」と表示する。
  推定で埋めない

## 構成要素

### 観測と制御
- レジスタ一式（シャドウ・**R**・IM/IFF込み。Rは本物: M1毎7bit増分＋
  bit7保持）。pause中はクリック編集（PC書き換え＝実行位置ジャンプ）
- 逆アセンブル（`z80dis.js`。Zilog形式デフォルト＋**Intel 8080形式**
  トグル — 88のモニタ文化準拠。Z80拡張はZilogのまま）
- メモリhexダンプ（編集可）、サブ側はFDC状態・モーターも
- ブレークポイント: アドレス＋**条件式**（レジスタ/`mem(addr)`を見る
  JS式、main/sub別）
- クロック常時表示: 総Tステート・実機換算時間（T/clockHz）・フレーム番号

### 時間旅行（無限undo/redo＋分岐ツリー）
- core: `machine.snapshot()`/`restore()`（8801はサブ基板・8255ペア・
  FDC込み。検証: test-snapshot.mjs）
- ICE: Nフレーム毎に自動スナップショット＋**入力イベントログ**
  （keyDown/keyUpをフレーム番号付きで記録）。undo＝最寄りノードへ
  restore→決定論再実行で正確に着地
- **分岐ツリー**: pause中に別の操作で再開すると子ブランチが生える。
  ノード＝スナップショット、クリックでジャンプ
- 正直な注記: マウント中のD88は参照保持（セクタ書き込みは巻き戻らない）

### プロファイラ
- シャドウコールスタック（CALL/RST/RET追跡）でルーチン毎に
  self T / total T / 呼び出し回数。実時間換算付き。シンボル名解決

### アセンブラ（`z80asm.js`）と静的解析（`z80anal.js`）
- 2パス・マクロ・`PROC USES`（自動push/pop）・`STRUC`（IXオフセット名）・
  `RELOC`（fixupテーブル出力）
- 解析: ルーチン毎の破壊/入力/保存レジスタ（CALL先へ推移的伝播）、
  I/Oポート使用（機種別ポート名注釈）、メモリアクセスマップ（既知領域名）、
  スタックバランスlint、Tステートmin/max、自己書き換え検出（⚡表示）

## 石器一式（リバースエンジニアリング工具）

ラベルを振り、ソースに起こし、別の場所に置き直す——firmware時代の
石器を、エミュレータ統合で「祈り」から「検証」に格上げする。

> 命名の由来: 作者の「機器」のtypo。あまりに的確だったので正式採用
> された（打製石器のように原始的で、無いと文明が始まらない工具群）。

1. **ラベルDB** — 逆アセン画面でアドレスに命名。localStorage保存、
   JSONエクスポート/インポート。z80asmのsymbolsと合流。逆アセン・
   プロファイラ・BP一覧すべてで名前解決
2. **ソース書き出し** — 範囲→ラベル付き逆アセン＋データ領域のDB/DW化
   →`ORG`付きソース。**再アセンブルで元バイト列と一致**（ラウンド
   トリップが受け入れ条件）
3. **リロケート** — 書き出したソースを別ORGで再アセンブル→書き込み。
   範囲内参照はラベル経由で自動追従
4. **トレースベース自動分離**（issue #5）— カバレッジマップ
   （M1実行/読み/書きをバイト単位）＋簡易taint（即値→アドレス使用の
   照合）で code/data/ポインタ を観測で確定。未踏は未分類のまま
5. **アドレス依存対策の3段構え**（issue #5追記）
   - **pin指定**: 範囲を移動対象から外す（ハード固定領域は自動pin）。
     「疑わしきは動かさない」の受け皿
   - **整列制約**: 上位/下位バイト単位のtaintで `LD H,imm` 型の
     上位バイトポインタを検出→256整列制約（0x100倍数シフトのみ許可
     or pin降格）
   - **双子実行diff**: 元とリロケート後を決定論エミュ2台で同入力並走
     →トレース比較。分岐＝見逃したアドレス依存。snapshot二分探索で
     分岐命令を特定（リロケートの正しさを検証にする）

## 依存の向き

```
ice.html ──wrap──▶ machine(.88).js ──▶ chips
   │
   ├─▶ z80dis.js（純関数）
   ├─▶ z80asm.js（純関数）──▶ z80anal.js
   └─▶ snap.js 経由の snapshot/restore（core提供）
```

UIはcoreを読むだけ。coreはUIを知らない。石器はすべて純粋モジュールで
headlessテスト可能——ICEはそれらの見せ方にすぎない。
