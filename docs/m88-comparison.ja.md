# upd3301 vs M88 — 挙動比較

`upd3301` が **M88**（`bubio/M88M`）とどう違うかの生きた記録。[`../m88ref/`](../m88ref/)
のヘッドレス参照神託で作る。手法: 同じ `.d88` を両方でヘッドレス・同フレーム数起動し、
fingerprint を比較する ── 最終 `E6CD`（ゲーム固有フラグ。多くのタイトルでキーボード
スキャンの門）、非ゼロテキストVRAMバイト数、ディスク読み活動。差分は「手がかり」で
あって「判決」ではない。各差分は命令単位で追う（テキストウィンドウのバグもこうやって
見つけた ── [io-ports](./io-ports.ja.md) と捜査記録参照）。

## スナップショット（250フレーム, m88204 ROMセット）

| タイトル | M88 E6CD / tvNZ | うち E6CD / tvNZ | 備考 |
|-------|-----------------|------------------|------|
| 軽井沢誘拐案内 | 00 / 3540 | 00 / 3571 | ✔ メニュー起動（テキストウィンドウ前は死亡） |
| Ys1 | 00 / 2678 | fc / 2678 | f250でE6CD差 — 要確認 |
| Hydlide | 00 / 2683 | 00 / 2684 | ✔ 一致 |
| Xanadu | f9 / 2673 | f9 / 2673 | ✔ 一致 |
| Romancia | 66 / 2678 | 66 / 2678 | ✔ 一致 |
| Thexder | 00 / 3416 | 00 / 3416 | ✔ 一致 |
| Dragon Buster | 00 / 3193 | 00 / 3193 | **⚠ 読みリトライループ**（下記） |
| Abyss2 | 00 / 3193 | 01 / 3194 | E6CD差 — 要確認 |
| 177 | 24 / 2679 | 24 / 3523 | E6CD一致・tvNZ差 |
| Again | ff / 2903 | ff / 3423 | E6CD一致（ffがこのタイトルの正常値） |
| Eldrad | ff / 3221 | ff / 3221 | 一致 |
| D-SIDE | ff / 3220 | ff / 3216 | 一致 |
| Aggres | ff / 3352 | ff / 3352 | 一致 |
| Asteka | 00 / 2700 | 00 / 2680 | ほぼ一致 |
| Argo | 09 / 2678 | 09 / 2678 | ✔ 一致 |

大半のタイトルが M88 の `E6CD`/tvram 状態に到達するようになった ── テキストウィンドウ
修正（port 70h）が大きな梃子だった。`E6CD == 0xff` は必ずしも「停止」ではない: 数本
（Again/Eldrad/D-SIDE/Aggres）は M88 でも 0xff を保つ。

### 注意: 読み数はまだ同一単位で比較できてない
うちのプローブは `READ DATA` **コマンド**（FDC op 0x06）を数える。refdrv の `g_rdN` は
`FDC::ReadData` 呼び出し数で、セクタ単位かもしれない。両者が同じイベントを数えるまで、
生の読み数は「等値テスト」ではなく「大まかな進捗指標」として扱う。

## 既知の差分（追うべき手がかり）

1. **Dragon Buster — 読みリトライループ。** うちは `cyl14 h1 r4` の SEEK→SENSE→READ を
   250フレームで**約17,000回**発行する（M88は数回読んで先へ進む）。読んだデータ/ステータス
   がタイトルのチェックを満たさない症状 ── プロテクト/検証読み、または FDC 結果バイト/`ST`
   の差。未根治。*(候補: 当該セクタのID/CRCステータス。)*
2. **Ys1 / Abyss2 — f250で `E6CD` 差**（fc/01 vs 00）。テキストVRAMは一致なので絵はほぼ
   正しい。フラグはタイミング位相か、実際のキーボード門差か。長時間実行＋画面ダンプで確認。
3. **177 — tvNZ が M88 より多い**（3523 vs 2679）、`E6CD` は一致。テキスト属性/塗り差の
   可能性。視覚確認が要る。
4. **Z80 未定義フラグ（X/Y, bit 3/5）。** 軽井沢のロード中の同一CPU状態で、うちとM88は
   文書化フラグ全一致だが未定義 X/Y ビットが違った。そのタイトルには無害（起動した）が、
   実在する Z80 忠実度のギャップ。一部プロテクトが側路でこれをテストする。監査価値あり。

## 再現/拡張手順

```sh
# M88側（一度ビルド）:
m88ref/build.sh
m88ref/_m88m_build/M88M/refdrv <romDir> <disk.d88> 250   # 最終 E6CD, tvramNZ, g_rdN を出力

# うち側: Nodeハーネス ── new Pc8801Machine → insertDisk → stepFrame ×250、
# その後 m.ram[0xe6cd] を読み、非ゼロ m.tvram[] を数え、globalThis.__fdcCmd をフック。
```
完全な手法と、貼るだけのサブエージェント prompt は [`../m88ref/README.ja.md`](../m88ref/README.ja.md)。
見つけた行/差分はここに追記していく。
