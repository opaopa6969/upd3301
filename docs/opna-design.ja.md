# サウンドボードII / YM2608 (OPNA) — 設計

いまエミュに載っている **YM2203 (OPN)** 音源を、拡張ボード **サウンドボードII**（チップは **YM2608 = OPNA**）へどう拡張するかの設計契約。既存の [`ym2203.js`](../ym2203.js) の構造に即して、**何を・どの順で作り・どう検証するか**を絵に描いた餅にせず書く。

## 1. なぜ — サウンドボードIIが足すもの

PC-8801mkII SR以降には内蔵の **YM2203 (OPN)**：FM×3 + SSG×3、モノラル。**サウンドボードII** は **YM2608 (OPNA)** を載せた拡張ボードで、上位互換：

| | 内蔵 OPN (YM2203) | サウンドボードII / OPNA (YM2608) |
|---|---|---|
| FM | 3声 | **6声** |
| SSG | 3声 | 3声（同一） |
| リズム | — | **ADPCM-A ×6**（バスドラ/スネア/シンバル/ハイハット/タム/リム）＝内蔵ROMの**本物のドラム** |
| PCM | — | **ADPCM-B ×1**（ボード上256KB RAMからの可変レートΔPCM） |
| 出力 | モノラル | **ステレオ**（FM各声にL/R） |
| タイマ | A/B | A/B（同一） |

Falcom作品（Ys, Xanadu, Sorcerian…）はブート時にボードを検出し、あれば **豪華版アレンジ**（FM6声＋ドラム）を、無ければOPN版を演奏する。OPNAを正しく作ることが、「TO MAKE THE END OF BATTLE」を素のFM3本じゃなく**ドラム入り**で鳴らす鍵。

## 2. すでにある資産（作り直さず再利用）

[`ym2203.js`](../ym2203.js) は肝心な部分がほぼOPNA対応済み：

- `class Operator` — DT/MUL/TL/KS/AR/DR/SR/RR/SL・phase・env・キー処理。**OPNAでも無改変。**
- `class FmChannel` — 4オペレータ, `alg`,`fb`,`fnum`,`block`。**無改変、6個にするだけ。**
- `_fmChannel(ch)` / `_opTick(op)` — FM合成コア（正統なLOG-SIN→EXP経路）。**既にチャンネル非依存**（`render()`は`this.ch`をループ）。
- `_writeSsg`,`_ssgTick`,`_ssgOut` — SSG。**無改変。**
- `tickTimers`・IRQ・`getState`/スナップショット — **無改変**、タイマは同一。
- 出力段（board模型・ライブツマミ）はそのまま、ステレオバスを食わせる形に。

つまりOPNAは大部分が **FMが増えるだけ** ＋ **本当に新規な2ユニット**（ADPCM-Aリズム, ADPCM-B PCM）＋ **ステレオ**。`Ym2608` を既存チップのサブクラス／上位集合として起こし、FM/SSGコードはそのまま共有する。

## 3. レジスタマップ（変わるところ）

OPNAは **2つのレジスタバンク** でアクセスする。PC-88では内蔵OPNがポート **44h(addr)/45h(data)**、サウンドボードIIは*別デバイス*のOPNAで **A8h(addr0)/A9h(data0+status)/AAh(addr1)/ABh(data1)** — 実測で確定（§9参照）。（初稿は46h/47hと推測したが誤り。）以下のレジスタ**内容**はどちらでも同じYM2608マップ。

**バンク0**（A8h/A9h）— いまのOPNマップの上位集合、`$00–$B6` は実装済み：
- `$00–$0F` SSG · `$22` LFO · `$24–$27` タイマA/B＋モード · `$28` FMキーオン/オフ · `$2D–$2F` プリスケーラ
- `$30–$9F` FMオペレータ params、ch**1–3** · `$A0–$A6` fnum · `$B0–$B2` FB/ALG · `$B4–$B6` **L/R＋AMS/PMS**（新規：ステレオpan＋LFO感度）

**バンク1**（46h/47h）— 全部新規：
- `$00–$0D` **ADPCM-A（リズム）**：`$00` キーオン/dump（bit0–5＝6つのドラム, bit7＝dump）, `$01` トータルレベル, `$08–$0D` ドラム毎のL/R＋レベル
- `$10–$1B` **ADPCM-B**：制御/キー, L/R, 開始/終了/リミットアドレス, ΔN(レート), レベル, DRAMアクセス
- `$30–$B6` FMオペレータ params、ch**4–6**（1–3と同レイアウト）

キーオンレジスタ `$28` の値は `bit0-1＝グループ内ch`, `bit2＝グループ(0⇒FM1–3, 1⇒FM4–6)`, `bit4-7＝オペレータslotマスク`。

## 4. 新規ユニットの設計

### 4.1 FM6声＋ステレオ
- `this.ch` を3→6 `FmChannel` に。`render()`は既に配列ループ、コア改変なし。
- `writeAddr`/`writeData` に **バンク** 概念を追加：バンク1アドレス書き込み(46h)は `this.addr1` にラッチ、バンク1データ(47h)は `$30–$B6` をch4–6へ、`$00–$1B` をADPCMへ振る。
- **ステレオ**：FM各声とリズム/PCM各声にL/Rイネーブル（`$B4–$B6`,`$08–$0D`,`$11`）。`render()`は `outL`/`outR` を生成、board出力段を左右で走らせる。OPNモードは両ビットON=モノ合成で従来と不変。

### 4.2 ADPCM-A — リズム（ドラム）
- 6固定声、PCM波形は小さな **内蔵リズムROM**（バスドラ/スネア/トップシンバル/ハイハット/タム/リムショット）に入っている。このROMは **外部アセット** として用意が要る（標準的なYM2608リズムROMダンプ、1音数KB）。PC-88のBIOS ROM同様、必須入力として明示する。
- デコード：YM2608 **ADPCM-A** は固定ステップテーブルの4bit ADPCM（フィルタ無し）。各アクティブドラムが自前のサンプルポインタをチップレートで進め、線形サンプルへデコード、ドラム毎レベル×トータルレベルでスケール、L/R pan、リズムバスへ加算。
- `$00` bit*n* 立ち上がり＝ドラム*n*トリガ（ポインタ再スタート）、bit7＝dump（停止）。決定論：同キー書き込み→同サンプル。

### 4.3 ADPCM-B — PCMチャンネル
- ボード256KB DRAMから読む **可変レートΔPCM** 1声。ゲームはサンプルをDRAMへDMA（`$08`/DRAMアクセスレジスタ）、開始/終了/リミット・ΔNピッチ・レベル・L/Rを設定してキー。
- デコード：YM2608 **ADPCM-B**（Aとは別の適応ステップADPCM）＋ΔN用の位相アキュムレータで出力レートへリサンプル。
- DRAMはただの `Uint8Array` 状態、`getState()` に含めて決定論スナップショット。

### 4.4 ボード検出
- ゲームはサウンドボードIIを探る（ステータス読み/ADPCM存在確認）。この探りに正しく応答して、ゲームに **豪華版スコアを選ばせる** 必要がある。これが無いと、完璧に鳴るOPNAでも6声＋ドラムのデータが来ない。**一級要件**、おまけではない。

## 5. 契約遵守（suite-contract）
- **pure/決定論**：`Math.random`なし、リズムROMとDRAMは静的データ、同レジスタ書き込み＋同ROM→同サンプル。スナップショットはADPCMポインタ＋DRAMを含む。
- **固定ステップ** `render(outL, outR, n)`、タイマ不変。
- **plain-data出力** ＋ `schemaVersion` 更新。
- **依存は一方向**：`ym2608.js`はcore（DOM/board非import）、`machine88.js`が44–47hを配線しステレオバッファを回す、demo/出力段は下流。共有FM/SSGコードは共通基底へ切り出しOPNとOPNAが乖離しないように。

## 6. 実装順（各フェーズが単独で聴ける/テストできる）

1. **FM6＋ステレオ＋バンク1ポート。** FM6声・L/R pan・`machine88`で46/47hデコード。SB2曲の多くはFM6主体なのでこれだけで厚くなる。*検証：* FM4–6のピッチ/アルゴリズムテスト（OPNテストを拡張）、ステレオpanテスト、決定論。
2. **ADPCM-Aリズム。** リズムROM読込・デコード・`$00–$0D`配線。*検証：* 各ドラムトリガ→非無音・相対レベル正・決定論、実SB2録音とA/B。
3. **ADPCM-B PCM。** DRAM＋デコーダ＋ΔNリサンプラ。*検証：* 既知ADPCM-Bブロックのゴールデンベクタ、スナップショット往復。
4. **ボード検出。** 探りに応答してゲームにSB2アレンジを選ばせる。*検証：* SB2対応Falcomをブートし6声＋ドラムスコア読込を確認（OPNモード比で楽器書き込み密度が跳ねる）。

## 7. 検証ハーネス（作ったものを再利用）
`opn-scope` のライブエンジンは、キャプチャしたレジスタ書き込みログをブラウザでチップに流し込んで再生する。キャプチャをバンク1書き込みへ、トレースを6 FM＋リズムのレーンへ拡張すれば、同じツールが **OPNAスコープ** になる：ドラムが鳴るのを見る、ADPCM-Bをソロ、同じゲームのOPN対OPNAをA/B。教師mp3比較のワークフローもそのまま使える。

## 8. 未解決/リスク
- **リズムROMの入手** — 必須外部アセット、BIOS ROM同様に同梱/サイドロードできるか確認。
- **ADPCM-B DMAタイミング** — ゲームがADPCM busyフラグ/IRQに依存しうる、デコーダだけでなくステータスビットの配線が要る。
- **検出のゲーム毎の差** — 探り方はタイトル/ドライバで違う、数本ブートをキャプチャして詰める必要あり。
- **出力段のステレオ化** — 現行board模型（HP・ソフトリミット・ライブツマミ）を左右に複製、ツマミUIの意味を二重化しないように。

## 9. 検証優先 — 敵対的レビュー＋実測I/O

独立レビュー（Codex）と、うちのエミュでのライブI/Oプローブにより、§3/§4 のいくつかの断定が「仮定」から「コード前に要確認」へ格下げになった。**検出が本丸**：ゲームにサウンドボードIIの存在を信じさせるまで、下流は全部デッドコード。

**実測（Ys III / Ys II をうちのマシンで20秒ブート、全 `in`/`out` をログ）：**
- 両ゲームともOPNモードでは音源として **0x44/0x45 しか触らない** — **0x46/0x47 や 0xA8–0xAF は読みも書きもしない**。多いのは 0xFC–0xFF（FDDサブCPUのPPIハンドシェイク）と 0xE4–0xE6（割込コントローラ）。
- 解釈：ゲームは **SB2無しと判断してOPN版を演奏、OPNA経路を一切叩かない**。誰も叩かない完璧なOPNAは無音。だから最初の成果物はFM6声ではなく、**検出を成功させること**、そしてゲームが第2バンクを書き始めるのを確認すること。

**実測＋逆アセンブルで解決（Telenet MUSICBOX音楽ディスク）：**
- **ポートマッピング — 確定：SB2 OPNA は `0xA8`(addr0)/`0xA9`(data+status0)/`0xAA`/`0xAB`(bank1)。** 46h/47hではない（§3初稿は誤り）。内蔵OPNは0x44/0x45のまま、OPNAは*別*デバイスで0xA8–0xAB。証拠 — ディスク自身のステータス読みヘルパを `IN 0xA9` で逆アセンブル：
  ```
  OUT (C),E        ; C=0xA8  OPNAレジスタ選択
  OR (HL) ×6       ; 整定ウェイト
  INC C            ; C: 0xA8 → 0xA9
  IN A,(C)         ; 0xA9 の OPNA ステータス読み
  ...
  LD A,(0x8FCE)    ; 記憶済みサウンドボードフラグを参照
  AND 03 / CP 03 / RET NZ
  LD C,0xA8        ; low2==3 なら音声出力を OPNA(0xA8) へ
  ```
- **検出ルーティング — ドライバは記憶フラグ**（ここでは RAM `0x8FCE` 下位2bit==3 ⇒ OPNA）で **OPN(0x44) と OPNA(0xA8) を切替**。フラグはブート時の 0xA9 検出読みで設定される。つまり「豪華版を鳴らす」＝「ブート時の 0xA9 ステータス読みに、実在OPNAが返す値を返させる」。

**検出 — クラック済み（実SB2ディスク `S.O.S. for SB2`）：**
- 最小OPNA(`sb2:true`)を0xA8–0xABに挿してSB2タイトルをブート。0xA9を1回読み、直後に `CP 01h` / `JP NZ`（命令単位でトレース）。ステータス返り値をスイープすると、**`0xA9 → 0x01` でOPNA検出＆レジスタ書き込み開始**、他の値(0x00,0x02,0x03,0x80…)はOPNパスへ。つまりSB2存在判定は**「0xA9のステータス == 0x01」**（bit0=Timer-Aオーバーフロー）。実OPNAは極小Timer-A＋ポーリングで到達、最小チップはこれを再現する必要（OPNAステータスにTimer-Aフラグを実装し、検出時に0x01を返す）。
- 検出が通ると `S.O.S.` はOPNAのSSG初期化($00-$0D)・モード($27)・キー($28)を書き、あとはメニューで待機。両SB2タイトル(`S.O.S. for SB2`, `神の聖都 DEMO SB2対応版`)は**メニュー/サウンドテスト型**：FM6/リズム音楽はキー操作後に始まる。完全キャプチャにはキーボード入力が必要（テキストVRAMを読んでメニューを見て、キー注入、$30-$B6/リズム書き込みを監視）。
- **リズムROM入手** ✅ — `OPNAリズムセット.zip` = ADPCM-Aドラム6音を44.1kHz/mono/16bit WAV(bd/sd/hh/rim/tom/top)で、`assets/opna-rhythm/`に保存。フェーズ2リズムはこれを直接再生できる（初版はADPCM-Aデコーダ不要）。

**残る未解決（[]内は方法）：**
1. **自然な検出ステータス** 🟠 — スタブでなく実Timer-Aフラグから0x01を返す。*[OPNAステータスにTimer-Aオーバーフロー実装、実チップでS.O.S.検出を確認]*
2. **音楽の完全キャプチャ** 🟠 — SB2タイトルのメニューをキーボード＋テキストVRAM読みで操作し再生開始、FM4-6＋リズムのレジスタ列=アレンジ参照を採る。*[RAM VRAMから画面読み、キーマトリクスマップ、操作、0xA8-0xAB書き込みログ]*
3. **ポートマップ** ✅解決 0xA8–0xAB。**検出** ✅クラック 0xA9==0x01。**リズムROM** ✅入手。
3. **`$28` キーオンのグループビット** 🟠 — 現行コードは `v & 3`（3ch）。OPNAは `bit2` でFM4–6を選ぶ必要があり、`& 3` は黙って落とす。*[OPNAモード時に bit2 を尊重するマスクに修正]*
4. **クロック/プリスケーラ** 🟠 — `fmStep = clk/72`, `ssgStep = clk/16` はOPNの分周。OPNAのプリスケーラ（`$2D–$2F`）と基底クロックは違いうる、6声だと数msのズレも可聴。*[データシートでOPNA分周を確認してから流用]*
5. **ADPCM-AリズムROM** 🟠 — 実ダンプ*と*そのフォーマット（4bit ADPCM、正確なステップテーブル、ドラム毎アドレステーブル）が要る。無いとデコーダすら書けない。*[YM2608リズムROM入手・フォーマット確認]*
6. **ADPCM-B DRAMレジスタ＋busy/IRQ** 🟠 — `$10–$1B` の割り当て（どれがアドレスでどれがデータ、オートインクリメント？）が未定、DMA後にbusyをポーリングするゲームはステータスビット未配線だと**ハングする**。*[ADPCM-B使用タイトルの $10–$1B 書き込み順をキャプチャ、デコーダだけでなくステータスビットを配線]*

**ビルド順(§6)への帰結：** フェーズ0は *「検出に到達する」* になった — マップしたポートにOPNA型のステータスを提示し、SB2対応ブートが豪華版スコアへ切り替わる（楽器書き込み署名が跳ねる）ことを確認する。それが済んで初めてFM6/リズム/ADPCM-Bが効いてくる。

## 10. フェーズ2実装 — 実Timer-A・リズム・マシン音声（＋fm.iplの足跡）

フェーズ0/1の骨組みを「鳴る・テスト済み」のOPNAにした回。検出フラグは**実タイマ**由来になり、**ドラムが鳴り**、**マシンの音声経路が1本**に。実SB2ディスクが要る分は最後に正直に切り分ける。

### 10.1 Timer-Aステータス＝実検出フラグ（スタブ無し）
SB2検出は `0xA9` を読んで `0x01`（§9）を期待する。この `0x01` は実タイマのオーバーフローから、**実機準拠のラッチ意味論**で出る（ハードコード無し）:
- `tickTimers()` はオーバーフローで status bit0 を**無条件に**立てる。`$27` bit2(enable) は `/IRQ` ラインだけをゲートし、フラグはゲートしない。（敵対的REレビューが旧「enableでフラグをゲート」を誤りと指摘して修正 — YM2612実機テストでフラグはenable無関係にラッチし、ドライバは割り込みを張らずにフラグをタイミング用にポーリングできる。）`$27` bit4 でクリア、`irq` ゲッターが各ラッチ済みフラグと対応enableビットをANDする。
- マシン経由でE2E検証: `0xA8/0xA9` で `$24/$25/$27` を書き、tick後 `machine.in(0xA9) === 0x01`。ラッチ vs IRQ の分離とリセットもテスト。`test-ym2608.mjs`（Timer-A群）と `test-opna-machine.mjs` を参照。これで検出は一段頑健に — SB2タイトルがIRQを張るか否かに関わらず `0x01` が出る。

`sb2:true` はチップ自身のタイマで検出を通る。実SB2タイトルが豪華版へ切替わる確認は§10.4（要ディスク）。

### 10.2 ADPCM-Aリズム — ドラムが鳴る
- `assets/opna-rhythm/*.wav`（6ドラム、44.1kHz/mono/16bit）を**デコード済みPCM**として `Ym2608.setRhythmRom(samples, rate)` で側方ロード（純チップはファイルに触らない）。`wav.js` は Node（`tools/load-rhythm-rom.mjs`）とブラウザ（`demo/machine.html`）両用の小さな純PCM-WAVデコーダ。ビット順は **BD SD TOP HH TOM RIM** ＝ `$00` bit0-5。
- `$00` 書き込み＝キー: bit *n* を bit7=0 で立てるとドラム *n* をリスタート、bit7=1 でダンプ。
- ドラム毎ゲインは `$01`(total,6bit)＋`$08–$0D`(個別,5bit) をチップの0.75dBステップで、`$08–$0D` の bit7/6 で L/R パン。ROMレートから出力レートへ線形補間。決定論（ポインタは `getState()` に入る）。
- リズムバスは `_aux()` フックで `render()`（モノ・ボード出力段経由）に合流、`renderRhythm(outL,outR)` はドラムのみをステレオで出す（A/B・テスト用）。テストは個別トリガ/パン/個別・総レベルの単調性/決定論/ボード経由の非無音を網羅。将来 `setRhythmRom()` の裏を本物のADPCM-Aデコーダに差し替え可能。

### 10.3 マシン音声経路を1本に
`Pc8801Machine.renderAudio(out, n)` が **OPN ＋（SB2なら）OPNA FM6＋リズム** を1呼び出しで描く。ドライバが叩いたチップだけが信号を出すのでOPN単体でもSB2でも加算は安全。`demo/machine.html` はこれを呼ぶ（SB2チェック時にリズムROMをfetch+デコードしてOPNAへ）。ブラウザ音声は依然**ヘッドレスでは視聴未検証**だが、mix経路はユニットテスト済み。

### 10.4 `bload"fm.ipl",r` → 「Internal error」の足跡（静的RE）
残る壁（issue手順1）は**実行時**の障害で再現にはディスクが要る。N88 ROMの静的逆アセンブルで狙いを絞った:

- **「Internal error」＝ N88-BASIC エラーコード 51 (0x33)。** メッセージ表は extバンク2（`n88_2.rom`、0x6000マップ）にあり "Internal error" は cpu **0x622B**、直後が "Bad file number"(52)・"File not found"(53) で MS-BASIC の 51/52/53 と一致。
- **BASICのERROR入口は main ROM `0x03B3`**（コードは `E` レジスタ）: `OUT (71h)` でメインバンク復帰→状態リセット→表示。
- **エラー51は extバンク3の cpu `0x67D3` で投出し:** `LD E,51 ; JP 0x03B3`。直前 `CALL 0x675B`(0x67D0) のチェックが失敗すると到達。
- 注目: その投出し周辺（**0x6756–0x67D8**）はOPN音源ポートを叩く — `OUT (44h)/OUT (45h)`（0x67B5のレジスタ書き込みヘルパ）、OPNレジスタ `$27`（タイマ制御）、`port 0x32`。つまり**サウンド経路**のルーチンで、`fm.ipl` がOPNAサウンドドライバである事と符合。（もう一つの `1E 33` 一致＝main 0x4DB5 は誤検出で、`LD BC,331Eh` ディスパッチ表の下位バイト。）

**SB2ディスクがある時の診断レシピ**（本repoはディスク画像を同梱しない — `roms/*` と `*.d88` はgit-ignore、CIでは走らない）: `sb2:true` でディスクをブートし、ICEで **0x67D3**（投出し）・**0x675B**（述語）・**0x03B3**（ハンドラ）にブレーク。`bload"fm.ipl",r` して最初に当たるブレークを見る。0x675B/0x67D3 なら述語の入力（読むRAMサウンド状態、0x44のOPNステータス）をダンプして何を弾いたか見る。どれも当たらなければ障害はbloadのヘッダ解析か `,R` ジャンプ先自身のOPNAアクセス — ブレークをbloadディスパッチャへ移す。

### 10.5 リズム/OPNAキャプチャ形式（§7スコープ＋参照グラブ用）
キャプチャ（要ディスク）は1書き込み1行のJSONで記録し `opn-scope` エンジンでOPNAスコープとして再生: `{ t, bank: 0|1, reg, val }`（bank0＝0xA8/0xA9、bank1＝0xAA/0xAB）。可視化レーン: FM1-6(`$B0` alg/fb)、リズム(`$00` key＋`$08–$0D` level/pan)、ADPCM-B(`$10–$1B`)。タイマ/キー($27/$28)でテンポ・ノートオンを刻む。

### 10.6 既知の近似（敵対レビュー注記）
独立したREレビュー（Codex）が上のTimer-A修正を追認し、以下を**正直な近似**（「まず鳴らす、精緻化は後」のスコープ通り、実SB2キャプチャ待ち）と指摘した:
- **リズムのレベルカーブ** — `_rhythmGain()` は total+個別を0.75dB/stepの加算アッテネータでモデル化。方向（レジスタ大→大音量）と単調性は正しいが、正確なtransfer curveは後で実機キャプチャにフィットしたLUTにすべき。
- **リズム＝デコード済みPCM（ADPCM-Aではない）** — 44.1kHzのドラムWAVを補間再生。本物のYM2608 ADPCM-A 4bitデコーダ（固定ステップ表・ドラム毎end address）は同じ `setRhythmRom()` の裏に入る。WAV経路はADPCM量子化のグリットを失う。
- **`$00` キー＝ソフトone-shot** — シリコンのキーラッチ/エッジ機ではない。bit7 dumpのエッジ挙動確定にはキャプチャが要る。
- **モノFM** — `ch.left/ch.right` は保持（`$B4-$B6`由来）だがレンダはモノ加算。片側ごとのボード段を通す完全ステレオFM6＋パン付きリズムは保留（issueはステレオ/ADPCM-Bを任意と明記）。
