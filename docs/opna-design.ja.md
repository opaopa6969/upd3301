# サウンドボードII / YM2608 (OPNA) — 設計

いまエミュに載っている **YM2203 (OPN)** 音源を、拡張ボード **サウンドボードII**（チップは **YM2608 = OPNA**）へどう拡張するかの設計契約。既存の [`ym2203.js`](../ym2203.js) の構造に即して、**何を・どの順で作り・どう検証するか**を絵に描いた餅にせず書く。

## 1. なぜ — サウンドボードIIが足すもの

PC-8801mkII SR以降には内蔵の **YM2203 (OPN)**：FM×3 + SSG×3、モノラル。**サウンドボードII** は **YM2608 (OPNA)** を載せた拡張ボードで、上位互換：

| | 内蔵 OPN (YM2203) | サウンドボードII / OPNA (YM2608) |
|---|---|---|
| FM | 3声 | **6声** |
| SSG | 3声 | 3声（同一） |
| リズム | — | **ADPCM-A ×6**（バスドラ/スネア/シンバル/ハイハット/タム/リム）＝内蔵ROMの**本物のドラム** |
| PCM | — | **ADPCM-B ×1**（ボード上256KB RAMからの可変レートΔPCM） |
| 出力 | モノラル | **ステレオ**（FM各声にL/R） |
| タイマ | A/B | A/B（同一） |

Falcom作品（Ys, Xanadu, Sorcerian…）はブート時にボードを検出し、あれば **豪華版アレンジ**（FM6声＋ドラム）を、無ければOPN版を演奏する。OPNAを正しく作ることが、「TO MAKE THE END OF BATTLE」を素のFM3本じゃなく**ドラム入り**で鳴らす鍵。

## 2. すでにある資産（作り直さず再利用）

[`ym2203.js`](../ym2203.js) は肝心な部分がほぼOPNA対応済み：

- `class Operator` — DT/MUL/TL/KS/AR/DR/SR/RR/SL・phase・env・キー処理。**OPNAでも無改変。**
- `class FmChannel` — 4オペレータ, `alg`,`fb`,`fnum`,`block`。**無改変、6個にするだけ。**
- `_fmChannel(ch)` / `_opTick(op)` — FM合成コア（正統なLOG-SIN→EXP経路）。**既にチャンネル非依存**（`render()`は`this.ch`をループ）。
- `_writeSsg`,`_ssgTick`,`_ssgOut` — SSG。**無改変。**
- `tickTimers`・IRQ・`getState`/スナップショット — **無改変**、タイマは同一。
- 出力段（board模型・ライブツマミ）はそのまま、ステレオバスを食わせる形に。

つまりOPNAは大部分が **FMが増えるだけ** ＋ **本当に新規な2ユニット**（ADPCM-Aリズム, ADPCM-B PCM）＋ **ステレオ**。`Ym2608` を既存チップのサブクラス／上位集合として起こし、FM/SSGコードはそのまま共有する。

## 3. レジスタマップ（変わるところ）

OPNAは **2つのレジスタバンク** でアクセスする。PC-88では内蔵OPNがポート **44h(addr)/45h(data)** = バンク0、OPNAの第2バンクがポート **46h(addr)/47h(data)** = バンク1。

**バンク0**（44h/45h）— いまのOPNマップの上位集合、`$00–$B6` は実装済み：
- `$00–$0F` SSG · `$22` LFO · `$24–$27` タイマA/B＋モード · `$28` FMキーオン/オフ · `$2D–$2F` プリスケーラ
- `$30–$9F` FMオペレータ params、ch**1–3** · `$A0–$A6` fnum · `$B0–$B2` FB/ALG · `$B4–$B6` **L/R＋AMS/PMS**（新規：ステレオpan＋LFO感度）

**バンク1**（46h/47h）— 全部新規：
- `$00–$0D` **ADPCM-A（リズム）**：`$00` キーオン/dump（bit0–5＝6つのドラム, bit7＝dump）, `$01` トータルレベル, `$08–$0D` ドラム毎のL/R＋レベル
- `$10–$1B` **ADPCM-B**：制御/キー, L/R, 開始/終了/リミットアドレス, ΔN(レート), レベル, DRAMアクセス
- `$30–$B6` FMオペレータ params、ch**4–6**（1–3と同レイアウト）

キーオンレジスタ `$28` の値は `bit0-1＝グループ内ch`, `bit2＝グループ(0⇒FM1–3, 1⇒FM4–6)`, `bit4-7＝オペレータslotマスク`。

## 4. 新規ユニットの設計

### 4.1 FM6声＋ステレオ
- `this.ch` を3→6 `FmChannel` に。`render()`は既に配列ループ、コア改変なし。
- `writeAddr`/`writeData` に **バンク** 概念を追加：バンク1アドレス書き込み(46h)は `this.addr1` にラッチ、バンク1データ(47h)は `$30–$B6` をch4–6へ、`$00–$1B` をADPCMへ振る。
- **ステレオ**：FM各声とリズム/PCM各声にL/Rイネーブル（`$B4–$B6`,`$08–$0D`,`$11`）。`render()`は `outL`/`outR` を生成、board出力段を左右で走らせる。OPNモードは両ビットON=モノ合成で従来と不変。

### 4.2 ADPCM-A — リズム（ドラム）
- 6固定声、PCM波形は小さな **内蔵リズムROM**（バスドラ/スネア/トップシンバル/ハイハット/タム/リムショット）に入っている。このROMは **外部アセット** として用意が要る（標準的なYM2608リズムROMダンプ、1音数KB）。PC-88のBIOS ROM同様、必須入力として明示する。
- デコード：YM2608 **ADPCM-A** は固定ステップテーブルの4bit ADPCM（フィルタ無し）。各アクティブドラムが自前のサンプルポインタをチップレートで進め、線形サンプルへデコード、ドラム毎レベル×トータルレベルでスケール、L/R pan、リズムバスへ加算。
- `$00` bit*n* 立ち上がり＝ドラム*n*トリガ（ポインタ再スタート）、bit7＝dump（停止）。決定論：同キー書き込み→同サンプル。

### 4.3 ADPCM-B — PCMチャンネル
- ボード256KB DRAMから読む **可変レートΔPCM** 1声。ゲームはサンプルをDRAMへDMA（`$08`/DRAMアクセスレジスタ）、開始/終了/リミット・ΔNピッチ・レベル・L/Rを設定してキー。
- デコード：YM2608 **ADPCM-B**（Aとは別の適応ステップADPCM）＋ΔN用の位相アキュムレータで出力レートへリサンプル。
- DRAMはただの `Uint8Array` 状態、`getState()` に含めて決定論スナップショット。

### 4.4 ボード検出
- ゲームはサウンドボードIIを探る（ステータス読み/ADPCM存在確認）。この探りに正しく応答して、ゲームに **豪華版スコアを選ばせる** 必要がある。これが無いと、完璧に鳴るOPNAでも6声＋ドラムのデータが来ない。**一級要件**、おまけではない。

## 5. 契約遵守（suite-contract）
- **pure/決定論**：`Math.random`なし、リズムROMとDRAMは静的データ、同レジスタ書き込み＋同ROM→同サンプル。スナップショットはADPCMポインタ＋DRAMを含む。
- **固定ステップ** `render(outL, outR, n)`、タイマ不変。
- **plain-data出力** ＋ `schemaVersion` 更新。
- **依存は一方向**：`ym2608.js`はcore（DOM/board非import）、`machine88.js`が44–47hを配線しステレオバッファを回す、demo/出力段は下流。共有FM/SSGコードは共通基底へ切り出しOPNとOPNAが乖離しないように。

## 6. 実装順（各フェーズが単独で聴ける/テストできる）

1. **FM6＋ステレオ＋バンク1ポート。** FM6声・L/R pan・`machine88`で46/47hデコード。SB2曲の多くはFM6主体なのでこれだけで厚くなる。*検証：* FM4–6のピッチ/アルゴリズムテスト（OPNテストを拡張）、ステレオpanテスト、決定論。
2. **ADPCM-Aリズム。** リズムROM読込・デコード・`$00–$0D`配線。*検証：* 各ドラムトリガ→非無音・相対レベル正・決定論、実SB2録音とA/B。
3. **ADPCM-B PCM。** DRAM＋デコーダ＋ΔNリサンプラ。*検証：* 既知ADPCM-Bブロックのゴールデンベクタ、スナップショット往復。
4. **ボード検出。** 探りに応答してゲームにSB2アレンジを選ばせる。*検証：* SB2対応Falcomをブートし6声＋ドラムスコア読込を確認（OPNモード比で楽器書き込み密度が跳ねる）。

## 7. 検証ハーネス（作ったものを再利用）
`opn-scope` のライブエンジンは、キャプチャしたレジスタ書き込みログをブラウザでチップに流し込んで再生する。キャプチャをバンク1書き込みへ、トレースを6 FM＋リズムのレーンへ拡張すれば、同じツールが **OPNAスコープ** になる：ドラムが鳴るのを見る、ADPCM-Bをソロ、同じゲームのOPN対OPNAをA/B。教師mp3比較のワークフローもそのまま使える。

## 8. 未解決/リスク
- **リズムROMの入手** — 必須外部アセット、BIOS ROM同様に同梱/サイドロードできるか確認。
- **ADPCM-B DMAタイミング** — ゲームがADPCM busyフラグ/IRQに依存しうる、デコーダだけでなくステータスビットの配線が要る。
- **検出のゲーム毎の差** — 探り方はタイトル/ドライバで違う、数本ブートをキャプチャして詰める必要あり。
- **出力段のステレオ化** — 現行board模型（HP・ソフトリミット・ライブツマミ）を左右に複製、ツマミUIの意味を二重化しないように。

## 9. 検証優先 — 敵対的レビュー＋実測I/O

独立レビュー（Codex）と、うちのエミュでのライブI/Oプローブにより、§3/§4 のいくつかの断定が「仮定」から「コード前に要確認」へ格下げになった。**検出が本丸**：ゲームにサウンドボードIIの存在を信じさせるまで、下流は全部デッドコード。

**実測（Ys III / Ys II をうちのマシンで20秒ブート、全 `in`/`out` をログ）：**
- 両ゲームともOPNモードでは音源として **0x44/0x45 しか触らない** — **0x46/0x47 や 0xA8–0xAF は読みも書きもしない**。多いのは 0xFC–0xFF（FDDサブCPUのPPIハンドシェイク）と 0xE4–0xE6（割込コントローラ）。
- 解釈：ゲームは **SB2無しと判断してOPN版を演奏、OPNA経路を一切叩かない**。誰も叩かない完璧なOPNAは無音。だから最初の成果物はFM6声ではなく、**検出を成功させること**、そしてゲームが第2バンクを書き始めるのを確認すること。

**実装前/実装中に潰す未解決項目（[]内は方法）：**
1. **検出プロトコル** 🔴 — SB2対応Falcomドライバが「SB2あり」をどう判定するか。うちのキャプチャでは専用ポートの受動プローブではない ⇒ OPNAで違うOPNステータスビット、DIP/設定バイト（0x30/0x31は実際読んでる）、あるいは検出時のみSB2ドライバを積むディスク、のいずれか。*[SB2版ディスクのブートI/Oをキャプチャ、OPN専用ブートと差分]*
2. **ポートマッピング** 🔴 — 作業仮説：SB2 = OPN基底のOPNA、バンク0=0x44/0x45、**バンク1=0x46/0x47**；0xA8–0xAFは*第2*ボードスロット（ここでは未使用）。チップ族としては整合するが実機PC-88配線での確証なし。*[MAMEのpc8801 OPNA接続/ハード保守マニュアルで確認]*
3. **`$28` キーオンのグループビット** 🟠 — 現行コードは `v & 3`（3ch）。OPNAは `bit2` でFM4–6を選ぶ必要があり、`& 3` は黙って落とす。*[OPNAモード時に bit2 を尊重するマスクに修正]*
4. **クロック/プリスケーラ** 🟠 — `fmStep = clk/72`, `ssgStep = clk/16` はOPNの分周。OPNAのプリスケーラ（`$2D–$2F`）と基底クロックは違いうる、6声だと数msのズレも可聴。*[データシートでOPNA分周を確認してから流用]*
5. **ADPCM-AリズムROM** 🟠 — 実ダンプ*と*そのフォーマット（4bit ADPCM、正確なステップテーブル、ドラム毎アドレステーブル）が要る。無いとデコーダすら書けない。*[YM2608リズムROM入手・フォーマット確認]*
6. **ADPCM-B DRAMレジスタ＋busy/IRQ** 🟠 — `$10–$1B` の割り当て（どれがアドレスでどれがデータ、オートインクリメント？）が未定、DMA後にbusyをポーリングするゲームはステータスビット未配線だと**ハングする**。*[ADPCM-B使用タイトルの $10–$1B 書き込み順をキャプチャ、デコーダだけでなくステータスビットを配線]*

**ビルド順(§6)への帰結：** フェーズ0は *「検出に到達する」* になった — マップしたポートにOPNA型のステータスを提示し、SB2対応ブートが豪華版スコアへ切り替わる（楽器書き込み署名が跳ねる）ことを確認する。それが済んで初めてFM6/リズム/ADPCM-Bが効いてくる。
